# PRD --- Система учета средств измерений

## 1. Цель проекта

Создать корпоративное веб-приложение для автоматизации учета средств
измерений (СИ) на предприятии Республики Казахстан...

## 2. Технологический стек

-   Backend: Laravel 12 (PHP 8.4) или ASP.NET Core 9
-   Frontend: React + Vite или Vue 3
-   PostgreSQL
-   REST API + JWT
-   Docker

## 3. Пользователи

-   Администратор
-   Главный метролог
-   Инженер-метролог
-   Руководитель подразделения
-   Просмотр (Read Only)

## 4. Функциональные требования

### 4.1 Реестр средств измерений

Хранить: - Инвентарный номер - Внутренний номер - Наименование - Тип -
Модель - Марка - Заводской номер - Производитель - Страна изготовления -
Год выпуска - Дата ввода в эксплуатацию - Подразделение - Ответственное
лицо - Место установки - Назначение - Диапазон измерений - Класс
точности - Цена деления - Предел допускаемой погрешности - Статус
(Работает, В ремонте, На поверке, На калибровке, Списан, Архив) - Фото -
Паспорт - Свидетельство поверки - Инструкция - Документы - Комментарии

### 4.2 Поверка

-   Дата поверки
-   Следующая поверка (автоматический расчет)
-   Организация
-   Номер свидетельства
-   Результат
-   Вложения

### 4.3 Калибровка

Аналогично поверке.

### 4.4 Ремонт

Дата, причина, исполнитель, стоимость, документы.

### 4.5 Списание

Причина, дата, акт, ответственный.

### 4.6 Справочники

Подразделения, ответственные лица, производители, организации по
поверке.

### 4.7 Журнал действий

Логирование создания, изменения, удаления, входа, выхода, импорта,
экспорта.

## 5. Dashboard

-   Общее количество СИ
-   По подразделениям
-   По статусам
-   Просроченные поверки
-   Поверка через 90/60/30/7 дней
-   Диаграммы и графики

## 6. Поиск и фильтрация

Полнотекстовый поиск и фильтрация по всем основным реквизитам.

## 7. Импорт и экспорт

Импорт: Excel, CSV.

Экспорт: Excel, CSV, PDF, Word, печать.

## 8. Уведомления

Email, Telegram, внутренние уведомления о сроках поверки.

## 9. Документы

Хранение PDF, DOCX, XLSX, JPG, PNG с версионностью.

## 10. Безопасность

HTTPS, JWT, RBAC, защита от SQL Injection, CSRF, XSS, резервное
копирование.

## 11. База данных

Нормализованная PostgreSQL: - ER-диаграмма - DDL - Индексы -
Ограничения - Внешние ключи

## 12. API

Полный REST API со Swagger/OpenAPI.

## 13. Интерфейс

Адаптивный дизайн, светлая и темная темы.

## 14. Дополнительные функции

-   QR-коды
-   Штрихкоды
-   Печать карточек
-   Массовое редактирование
-   Архивирование

## 15. Отчеты

Поверки, просрочки, подразделения, производители, статистика, PDF/Excel.

## 16. Производительность

Поддержка не менее 100 000 записей, пагинация, кэширование, оптимизация
SQL.
